# Premium Frontend Page Playbook

## When to Use

Use this playbook for tasks such as:

- 高级前端页面
- landing page
- portfolio
- SaaS homepage
- 产品官网
- 个人简历网站
- 高质感 Web UI

Do not use it for backend-only work or raw asset curation.

## Task Intake Checklist

- Page type
- Target user
- Content structure
- Visual direction
- Required brand or domain cues
- Required assets or motion references
- Responsive requirements
- Animation or interaction expectations
- Real metrics versus placeholders

## Execution Steps

1. Determine page type and visual direction.
2. Select the layout system.
3. Select typography and spacing rhythm.
4. Select the component system.
5. Decide asset usage and usage policy.
6. Implement responsive structure.
7. Add interaction states and motion only where they support the task.
8. Verify readability, hierarchy, asset policy, and performance.

## Must Include in Output

- Visual direction
- Layout system
- Component list
- Interaction and motion policy
- Asset usage policy
- Rules or topic files used
- Verification notes
- Remaining uncertainty

## Avoid

- Using glassmorphism only to signal "premium"
- Applying video hero patterns to every page
- Sacrificing readability for visual effects
- Using assets without checking `usage_policy`
- Treating brand flavor references as core rules
- Filling the page with generic cards or placeholder metrics

## Completion Check

- The first viewport has a clear visual anchor.
- Headings, buttons, and cards do not overlap.
- Mobile and desktop layouts are both readable.
- Assets are either direct-use or explicitly inspiration-only.
- The final answer names the rule files that shaped the result.
