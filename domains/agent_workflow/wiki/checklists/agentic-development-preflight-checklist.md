# Agentic Development Preflight Checklist

> 来源：`garrytan/gstack` (MIT) — Think→Plan 阶段前置检查
> 生成时间：2026-04-30
> 使用时机：Agent 开始任何开发任务前，必须先过一遍此清单

## 一、需求明确性检查

- [ ] 我理解用户要解决什么问题？（不是"用户要什么功能"）
- [ ] 用户明确说了想要什么？还是我需要先问清楚？
- [ ] 这个问题可以用非代码方式解决吗？（配置、现有工具、流程改进）
- [ ] 最简单的 MVP 是什么？能否在 30 分钟内验证？

## 二、上下文获取检查

- [ ] 是否需要调用 `/brief` 获取数据库知识？
  - [ ] 如果是 UI/前端：`--ui 8 --workflow 2`
  - [ ] 如果是后端/API：`--backend 8 --workflow 2`
  - [ ] 如果是浏览器自动化：`--automation 8 --workflow 4`
  - [ ] 如果是 AI agent 工作流：`--workflow 8`
- [ ] API 不可用时，是否已读取对应 domain 的 README？
- [ ] 是否已读取相关的 rules 文件？

## 三、计划审查检查

- [ ] 是否需要 Plan 阶段？（新功能/大重构→必须；单行修复→跳过）
- [ ] 如果需要 Plan：
  - [ ] 架构设计是否已输出？
  - [ ] 数据流是否清晰？
  - [ ] 测试策略是否明确？
  - [ ] 范围是否已最小化？（CEO Gate）
  - [ ] 是否已做过至少一种审查？

## 四、技术栈与约束检查

- [ ] 项目运行在什么平台？（Windows/Linux/macOS/Web）
- [ ] 有什么已知的平台限制？
- [ ] 有什么不可用的依赖？（Go? Rust? Docker?）
- [ ] 是否需要联网？如果需要，是否有 fallback？
- [ ] 有没有许可证冲突？
- [ ] 有没有需要特别处理的安全边界？（认证/支付/用户数据/公网暴露）

## 五、质量闸门检查

- [ ] 是否需要代码审查？（非单行修改→必须）
- [ ] 是否需要测试？（新功能→必须）
- [ ] 是否需要浏览器 QA？（Web/UI 项目→建议）
- [ ] 是否需要安全审计？（认证/支付/用户数据→必须）
- [ ] 是否需要文档更新？（API 变更→必须）

## 六、发布前检查

- [ ] 是否所有测试通过？
- [ ] 是否所有审查闸门已完成？
- [ ] 是否有未处理的 P0 问题？
- [ ] 是否需要金丝雀监控？（生产部署→必须）
- [ ] 是否已记录 Retro？

## 七、Agent 自检

- [ ] 我有没有直接跳到 Build？（应该先 Think → Plan）
- [ ] 我有没有跳过边界情况？（Boil the Lake：做完整的事）
- [ ] 我有没有先搜索现有方案？（Search Before Building）
- [ ] 我有没有把推荐当作决定？（User Sovereignty：AI 推荐，用户决定）
- [ ] 我有没有记录这次的经验？（Retro：写入数据库）
